## 1. 问题描述

### 1.1 用户反馈
用户上传视频后，前端进度长时间显示"导入中"（字幕生成阶段），根据日志分析，本次处理中：

- **视频时长**: 约11分48秒（708秒）
- **字幕生成耗时**: 约5分22秒（322秒）
- **实时率**: 约2.19x（音频时长/处理时长）

### 1.2 进度显示问题
前端轮询机制每2秒查询一次进度，但字幕生成阶段没有细粒度的进度更新，导致用户感知卡顿。

## 3. 原因分析

### 3.1 主要瓶颈

```
处理时间分解（1小时视频）:
┌─────────────────────────────────────────────────────┐
│ 语音转写   ████████████████████░░░░░░░  ~35分钟    │  ← 主要瓶颈 (75-80%)
│ 字幕生成   █░░░░░░░░░░░░░░░░░░░░░░░░░░  ~2分钟     │
│ 其他处理   ███░░░░░░░░░░░░░░░░░░░░░░░░  ~8分钟     │
└─────────────────────────────────────────────────────┘
```

### 3.2 根因分析

| 因素 | 影响 | 说明 |
|------|------|------|
| **CPU推理** | 高 | 当前使用CPU，GPU未启用 |
| **模型大小** | 中 | Paraformer-zh约200MB，推理仍需时间 |
| **音频长度** | 高 | 11分钟音频需要完整处理 |
| **缺乏进度反馈** | 中 | 用户感知差，无细粒度进度 |
| **VAD未启用** | 中 | 静音片段仍被处理 |

### 3.3 进度显示问题分析

当前进度更新机制：
```
SUBTITLE阶段权重: 15%
进度范围: 10% - 25%
问题: 整个字幕生成都在这15%范围内，无细分
```

## 4. 优化方案评估

### 4.1 方案矩阵

| 方案 | 实施难度 | 预期收益 | 优先级 | 状态 |
|------|---------|---------|--------|------|
| **GPU加速** | 低 | 5-7倍速度提升 | **P0** | ✅ 已实现 |
| **模型量化** | 低 | 内存降低75%，速度提升2-3倍 | **P0** | ✅ 已实现 |
| **VAD预处理** | 中 | 30-50%时间节省 | P1 | 📋 设计完成 |
| **细粒度进度反馈** | 低 | 用户体验提升 | P1 | 📋 待实现 |
| **异步处理** | 中 | 后台处理，不阻塞UI | P1 | 📋 待实现 |
| **云端API** | 高 | 生产级性能 | P3 | 📋 设计完成 |

### 4.2 方案详细分析

#### 方案一：GPU加速（推荐）

**原理**: 利用CUDA/MPS设备进行并行计算

**预期效果**:
- 速度提升: 5-7倍
- 11分钟视频处理时间: ~45-65秒

**实施方式**:
```bash
# 设置环境变量
SPEECH_DEVICE=cuda
```

**当前状态**: 已实现自动检测，但当前环境未检测到GPU

---

#### 方案二：VAD预处理

**原理**: 使用VAD（语音活动检测）跳过静音片段

**预期效果**:
- 时间节省: 30-50%（取决于静音比例）
- 典型提升: ~40%

**实施方式**: 在语音识别前先进行VAD检测，只处理有语音的片段

---

#### 方案三：细粒度进度反馈

**原理**: 在字幕生成过程中增加进度更新点

**实施方式**:
- 按时间分段报告进度（每30秒更新一次）
- 或按处理百分比报告（10%, 20%, ... 100%）

**预期效果**: 提升用户体验，避免感知卡顿

---

#### 方案四：异步处理

**原理**: 将字幕生成放在后台任务处理

**实施方式**:
- 上传后立即返回，后台异步生成字幕
- 通过WebSocket推送进度更新

**预期效果**: 用户无需等待，可继续其他操作

## 5. 推荐方案

### 5.1 短期方案（立即生效）

1. **启用GPU加速**（如果有GPU）
   ```bash
   SPEECH_DEVICE=cuda python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
   ```

2. **增加细粒度进度反馈**
   - 修改 `_generate_subtitle_funasr()` 函数，增加进度回调

### 5.2 中期方案（1-2周）

1. **实现VAD预处理**
   - 使用FunASR内置的VAD模型
   - 跳过静音片段，只处理语音部分

2. **实现异步处理**
   - 任务提交后立即返回
   - 通过WebSocket实时推送进度

### 5.3 长期方案（1-2月）

1. **集成云端API**（阿里云、腾讯云等）
2. **实现批量处理**
3. **增加性能监控**
