# 时间重叠修复记录

## 问题描述

在视频处理流程中，发现话题时间线存在重叠问题，导致视频预览时间与实际时间不一致。

### 根本原因

1. **LLM 生成时间线时未确保话题之间时间不重叠**
2. **验证逻辑只检查时间格式和在块范围内，不检查重叠**
3. **物理分块导致话题可能在边界被切断**

### 问题示例

```json
[
  {
    "outline": "地域人群特征与文化影响",
    "start_time": "00:00:00,210",
    "end_time": "00:07:59,493"
  },
  {
    "outline": "其他讨论与闲聊",
    "start_time": "00:00:00,210",
    "end_time": "00:00:42,773"
  }
]
```

两个话题都从 `00:00:00,210` 开始，导致时间重叠。

---

## 修复方案

### 阶段一：时间重叠检测与修正（已完成）

**目标**：在 LLM 生成时间线后，检测并修正重叠的时间段

**文件**：`backend/pipeline/step2_timeline.py`

**修改内容**：

#### 1. 添加 `_fix_overlapping_times` 方法

```python
def _fix_overlapping_times(self, items: List[Dict]) -> List[Dict]:
    """
    修复时间重叠问题

    策略：按 start_time 排序后，如果后一个话题的 start_time < 前一个话题的 end_time，
    则将后一个话题的 start_time 调整为前一个话题的 end_time。
    如果修正后时长过短（<5秒），则跳过该话题。
    """
```

**处理逻辑**：
1. 按 `start_time` 排序所有时间段
2. 检测相邻话题是否时间重叠
3. 如果重叠，将后一个话题的 `start_time` 调整为前一个话题的 `end_time`
4. 如果修正后时长过短（<5秒），跳过该话题
5. 如果话题被完全包含，跳过该话题

#### 2. 添加 `_seconds_to_srt_time` 方法

```python
def _seconds_to_srt_time(self, seconds: float) -> str:
    """
    将秒数转换为SRT时间格式 (HH:MM:SS,mmm)
    """
```

#### 3. 在 `_parse_and_validate_response` 中调用修复方法

```python
# 在 return validated_items 之前
validated_items = self._fix_overlapping_times(validated_items)
```

## 遗留问题

### 问题

"其他讨论与闲聊"（钩子内容）被完全包含在"地域人群特征与文化影响"中而被跳过。

### 原因

LLM 在提取时间时判断错误：
- **正确**：闲聊 00:00:00,210→00:00:42,773，地域人群 00:00:42,773→00:07:59,493
- **LLM 错误**：闲聊 00:00:00,210→00:00:42,773，地域人群 00:00:00,210→00:07:59,493

### 解决方案

**阶段二：滑动窗口重叠** - 通过在块边界添加重叠区域，减少话题被切断的问题。
