## 1. 环境自动检测方案

### 1.1 方案概述

实现运行环境自动检测，根据检测结果自动选择最优配置策略，实现本地部署和云服务器部署的兼容。

### 1.2 核心功能

| 功能 | 说明 | 实现状态 |
|------|------|---------|
| **本地环境检测** | 识别本地/服务器环境 | ✅ 已实现 |
| **SSD检测** | 自动识别存储类型 | ✅ 已实现 |
| **资源检测** | 获取CPU核心数和可用内存 | ✅ 已实现 |
| **智能配置生成** | 根据环境自动选择最优配置 | ✅ 已实现 |
| **手动覆盖** | 支持环境变量手动设置 | ✅ 已实现 |

### 1.3 配置策略矩阵

| 环境类型 | chunk_size | max_workers | use_hardlink | io_delay |
|---------|------------|-------------|--------------|----------|
| **本地SSD** | 8MB | max(4, min(cpu, mem/2)) | True | 0s |
| **本地HDD** | 512KB | min(2, cpu) | True | 0.1s |
| **服务器** | 2MB | min(4, cpu) | False | 0.05s |

### 1.4 实现文件

- **核心类**: `backend/utils/environment_detector.py`
- **测试脚本**: `test_environment_detection.py`
- **设计文档**: `docs/environment_detection_design.md`

### 1.5 使用方式

```python
from backend.utils.environment_detector import EnvironmentDetector

# 获取环境配置
config = EnvironmentDetector.get_optimal_config()

# 在上传和切片中使用配置
chunk_size = config["chunk_size_bytes"]
max_workers = config["max_workers"]
```

### 1.6 环境变量配置

```bash
# 手动指定环境类型
ENVIRONMENT=local    # 本地环境
ENVIRONMENT=server  # 服务器环境

# 指定计算设备
SPEECH_DEVICE=cuda   # GPU加速
SPEECH_DEVICE=cpu    # CPU模式
```

## 2. 语音转写优化方案

### 2.1 现状分析

| 组件 | 当前方案 | 状态 |
|------|---------|------|
| **主引擎** | FunASR | ✅ 默认启用 |
| **备选引擎** | Whisper | ✅ 回退方案 |
| **辅助引擎** | bcut-asr | ⚠️ 可选安装 |

### 2.2 性能瓶颈

```
1小时视频处理时间分解:
┌─────────────────────────────────────────────────────┐
│ 语音转写   ████████████████████░░░░░░░  ~35分钟    │
│ 字幕生成   █░░░░░░░░░░░░░░░░░░░░░░░░░░  ~2分钟     │
│ 其他处理   ███░░░░░░░░░░░░░░░░░░░░░░░░  ~8分钟     │
└─────────────────────────────────────────────────────┘
              语音转写占总耗时 75-80% ⚠️ 主要瓶颈
```

### 2.3 优化方案

#### 方案一：GPU加速（已实现）

- 自动检测CUDA/MPS/CPU
- 支持环境变量手动指定
- 预期提升：5-7倍

#### 方案二：模型量化（已实现）

- INT8量化减少内存占用
- 内存降低75%
- 速度提升2-3倍

#### 方案三：VAD预处理（设计完成）

- 跳过静音片段
- 预期提升：30-50%

#### 方案四：异步并行处理（设计完成）

- 多视频批量处理
- 预期提升：3-4倍

### 2.4 实现文件

- **核心修改**: `backend/utils/speech_recognizer.py`
- **测试脚本**: `test_speech_optimization.py`, `test_speech_performance.py`
- **设计文档**: `docs/speech_recognition_optimization_design.md`

### 2.5 语音识别性能对比

| 方法 | 平均耗时 | 最快 | 最慢 | 成功率 |
|------|---------|------|------|--------|
| **funasr** | **9.42秒** | 1.25秒 | 25.76秒 | 100% |

## 3. 模型预加载功能

### 3.1 功能概述

在应用启动时提前加载语音识别模型，避免首次调用时的延迟。

### 3.2 实现细节

#### 预加载顺序

1. **FunASR模型**（默认引擎）
   - paraformer-zh（语音识别）
   - fsmn-vad（语音活动检测）
   - ct-punc（标点恢复）

2. **Whisper模型**（回退引擎）
   - small模型（平衡速度和准确率）

#### 特性

| 特性 | 说明 |
|------|------|
| **自动设备检测** | 自动检测CPU/GPU/MPS |
| **模型量化** | 默认启用INT8量化 |
| **缓存机制** | 按设备类型缓存 |
| **优雅降级** | 预加载失败不影响服务启动 |
| **后台加载** | 异步执行，不阻塞启动 |

### 3.3 实现文件

- **修改文件**: `backend/main.py`
- **测试脚本**: `test_preload_verify.py`

### 3.4 性能提升效果

| 场景 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 用户首次请求 | ~27秒 | ~1.3秒 | **21倍** |
| 后续请求 | ~1.3秒 | ~1.3秒 | 无变化 |
