## 一、项目背景

### 1.1 当前问题

| 问题类别 | 具体问题 | 影响 |
|---------|---------|------|
| **方言识别** | 济宁方言识别准确率不高 | 用户体验下降 |
| **模型性能** | 内存占用较高，普通PC运行困难 | 部署受限 |
| **处理速度** | Step 6视频导出耗时较长 | 等待时间长 |

### 1.2 目标

通过引入 `iic/speech_paraformer-large-vad-punc-zh` 模型及相关优化，解决上述问题，提升产品竞争力。

## 三、各阶段详细计划

### 阶段 1：模型优化升级（预计 1-2 周）

#### 目标
- 替换现有语音识别模型为 Paraformer
- 添加方言处理支持
- 支持模型量化，降低内存占用

#### 任务清单

| 序号 | 任务名称 | 描述 | 交付物 |
|------|---------|------|--------|
| 1.1 | 依赖安装 | 安装 funasr、modelscope 等依赖 | 更新 requirements.txt |
| 1.2 | 模型集成 | 修改 speech_recognizer.py 集成 Paraformer | 新的语音识别工具类 |
| 1.3 | 量化配置 | 支持 INT8/FP16 量化模式 | 配置文件 |
| 1.4 | 方言词典 | 创建济宁方言词汇映射表 | dialect_dict.json |
| 1.5 | 后处理模块 | 实现方言词汇替换逻辑 | 方言处理模块 |
| 1.6 | 测试验证 | 对比不同配置下的识别准确率 | 测试报告 |

#### 技术方案

**配置文件结构** (`config/asr_config.yaml`)：
```yaml
asr:
  model_type: "paraformer"
  model_name: "iic/speech_paraformer-large-vad-punc-zh"
  model_revision: "v2.0.4"
  quantize: true
  device: "cpu"
  
  dialect:
    enabled: true
    region: "jining"
    custom_dict: "config/dialect_dict.json"
```

**方言词典结构** (`config/dialect_dict.json`)：
```json
{
  "name": "济宁方言词典",
  "version": "1.0",
  "entries": {
    "白搭": "没用",
    "甭提": "别提",
    "老丝儿": "老师",
    "忒": "太",
    "稀罕": "喜欢",
    "咋": "怎么",
    "揍嘛": "干什么",
    "叨叨": "唠叨",
    "各应": "讨厌",
    "憨熊": "傻瓜"
  }
}
```

### 阶段 2：性能优化（预计 2-3 周）

#### 目标
- 优化 Step 6 视频导出速度
- 集成 GPU 加速支持
- 实现并行处理

#### 任务清单

| 序号 | 任务名称 | 描述 | 预期效果 |
|------|---------|------|---------|
| 2.1 | VAD 优化 | 一次检测，多次复用结果 | Step 6 减少 10-15s |
| 2.2 | 并行处理 | 多线程同时处理多个切片 | Step 6 减少 20-25s |
| 2.3 | GPU 加速 | 支持 CUDA 推理 | 识别速度提升 5-10x |
| 2.4 | 缓存策略 | 模型热加载，减少启动时间 | 启动时间减少 |
