## 核心改进详解

### 改进1: 轻量化说话人识别 - `simple_speaker_recognizer.py`

**特点：**
- 零外部依赖，纯 Python 实现
- 基于文本特征聚类（K-means，K=2）
- 特征包括：文本长度、标点习惯、平均词长、数字出现频率、感叹词
- 保留与原方案相同的接口
- 完整的缓存机制

**优势：**
- ✅ 部署简单，无需额外安装
- ✅ 运行速度快，无需加载大模型
- ✅ 适合快速验证
- ✅ 接口兼容，未来可随时替换为 CAM++

### 改进2: 双重提示词策略 - Step2完全集成

借鉴 FunClip 的核心策略：

**阶段1: 内容理解**
- 调用 `_do_content_understanding()` 方法
- 使用 `content_understanding.txt` 提示词
- 让 LLM 先分析 SRT 的话题结构和标志性开头
- 输出结构化的 JSON

**阶段2: 增强定位**
- 将阶段1的分析结果注入到原时间线提示词中
- 特别强调标志性开头
- 使用 `_build_enhanced_timeline_prompt()` 构建增强提示词

**优势：**
- ✅ 提高边界准确性
- ✅ 自动识别标志性开头（"京油子"等）
- ✅ 保持话题完整性

### 改进3: 优化集成顺序

**原方案问题：**
- 说话人识别在最后一步（时间线已完成）
- 无法利用说话人信息优化时间定位

**方案A优化：**
- 说话人识别前置（在第0步完成）
- 虽未直接用于提示词，但更符合逻辑流程
- 后续可扩展，将说话人信息注入提示词
