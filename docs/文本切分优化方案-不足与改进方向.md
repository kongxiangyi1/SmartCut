## 当前方案不足分析

### 1. 关键词提取过于简单 ❌

#### 当前问题
```python
def _extract_keywords(text: str) -> List[str]:
    # 简单按空格和标点切分
    words = re.split(r'[\s,，]', text)
    # 只保留2字以上的词
    keywords = [w.strip() for w in words if len(w.strip()) >= 2]
```

#### 具体问题
| 问题 | 影响 | 示例 |
|------|------|------|
| 没有中文分词 | 切分效果差 | "今天天气好" → ["今天天", "天气好"] |
| 没有去除停用词 | 噪音多 | ["的", "了", "是", "在"] 都算关键词 |
| 没有考虑词频 | 重要性不分 | 专业术语和常见词权重一样 |
| 无法识别专业术语 | 准确性低 | "卷积神经网络" → 可能被切分成多个词 |

#### 改进方案
```python
# 方案A：引入 jieba 分词（推荐）
import jieba
import jieba.analyse

def _extract_keywords_improved(text: str, top_k: int = 10) -> List[str]:
    # 使用 TF-IDF 提取关键词
    keywords = jieba.analyse.extract_tags(
        text, 
        topK=top_k, 
        withWeight=True,
        allowPOS=('n', 'vn', 'v')  # 只保留名词、动词等
    )
    
    # 提取关键词（可以考虑权重）
    return [word for word, weight in keywords]

# 方案B：轻量级停用词过滤（不需要额外依赖）
def _extract_keywords_light(text: str) -> List[str]:
    # 简单停用词列表
    stop_words = {"的", "了", "是", "在", "有", "和", "就", "都", "而", "及", "与", 
                  "着", "或", "一个", "没有", "我们", "你们", "他们"}
    
    # 简单但更有效的分词
    words = []
    # 按常见分隔符切分
    for seg in re.split(r'[，。！？、\s]', text):
        # 对于长文本，尝试简单切分
        if len(seg) > 4:
            words.append(seg[:2])  # 前2字
            words.append(seg[-2:])  # 后2字
        words.append(seg)
    
    # 过滤停用词和短词
    keywords = [w for w in words if len(w) >= 2 and w not in stop_words]
    return list(set(keywords))  # 去重
```

#### 当前问题
```python
# 简单关键词重叠度
overlap = len(keywords_before & keywords_after) / len(keywords_before | keywords_after)
if overlap < 0.3:
    # 认为是语义边界
```

#### 具体问题
| 问题 | 说明 |
|------|------|
| 只看关键词重叠 | "AI"和"人工智能" → 0重叠，但语义相同 |
| 不考虑语序和上下文 | "今天天气好" vs "好天气今天" → 关键词一样，但意思不同 |
| 0.3的阈值固定 | 可能不适合所有场景（如专业内容 vs 闲聊） |

#### 改进方案
```python
def _calculate_semantic_similarity(text1: str, text2: str) -> float:
    """
    更准确的语义相似度计算
    
    方案A：使用预训练模型（如Sentence-BERT）
    方案B：简单但更有效的规则 + 关键词重叠
    """
    
    # 方案：多级相似度计算
    score = 0.0
    
    # 1. 关键词重叠（基础分）
    keywords1 = set(_extract_keywords(text1))
    keywords2 = set(_extract_keywords(text2))
    if keywords1 & keywords2:
        overlap = len(keywords1 & keywords2) / len(keywords1 | keywords2)
        score += overlap * 0.4
    
    # 2. 字符相似度（对于没有关键词的情况）
    from difflib import SequenceMatcher
    char_sim = SequenceMatcher(None, text1, text2).ratio()
    score += char_sim * 0.3
    
    # 3. 主题词匹配（如果有主题词库）
    # ...
    
    # 4. 长度差异惩罚
    len_ratio = min(len(text1), len(text2)) / max(len(text1), len(text2))
    score += len_ratio * 0.3
    
    return score

# 使用：
```

### 3. 没有考虑文本密度差异 ❌

#### 当前问题
```python
# 只看时间
interval_minutes = 30
```

#### 具体问题
| 视频类型 | 内容密度 | 30分钟内容量 |
|---------|---------|------------|
| 教学类 | 高（10字/秒） | ~18000字 |
| 闲聊类 | 低（2字/秒） | ~3600字 |
| 电影/电视剧 | 中等 | ~10000字 |

#### 改进方案
```python
def _calculate_optimal_chunk_duration(srt_data: List[Dict]) -> int:
    """
    根据文本密度动态计算最优切分时长
    
    返回：分钟数
    """
    # 1. 计算文本密度（每秒字数）
    total_text = " ".join([sub['text'] for sub in srt_data])
    total_chars = len(total_text)
    
    # 计算总时长
    if srt_data:
        start = text_processor.time_to_seconds(srt_data[0]['start_time'])
        end = text_processor.time_to_seconds(srt_data[-1]['end_time'])
        total_seconds = end - start
    else:
        total_seconds = 1800  # 默认30分钟
    
    chars_per_second = total_chars / total_seconds if total_seconds > 0 else 5
    
    # 2. 根据密度调整
    if chars_per_second > 8:
        # 高密度内容（教学类）→ 15-20分钟
        return 15
    elif chars_per_second > 4:
        # 中等密度 → 25-30分钟
        return 30
    else:
        # 低密度内容（闲聊）→ 30-45分钟
        return 45
```

### 5. 性能优化不足 ❌

#### 当前问题
| 问题 | 说明 |
|------|------|
| 每次重新计算停顿点 | 对于超长视频，浪费时间 |
| 没有缓存 | 如果多次处理同一段视频，重复计算 |
| 处理效率低 | 对于1000条以上的字幕，可能较慢 |

#### 改进方案
```python
class TextProcessor:
    def __init__(self):
        self._pause_cache = {}  # 缓存停顿分析结果
        self._chunk_cache = {}   # 缓存切分结果
    
    def chunk_srt_data_cached(self, srt_data: List[Dict], ...) -> List[Dict]:
        """
        带缓存的切分函数
        """
        # 生成缓存键（基于内容哈希）
        cache_key = hash(str(srt_data) + str(interval_minutes))
        
        if cache_key in self._chunk_cache:
            logger.debug("使用缓存的切分结果")
            return self._chunk_cache[cache_key]
        
        # 计算并缓存
        chunks = self.chunk_srt_data(srt_data, ...)
        self._chunk_cache[cache_key] = chunks
        return chunks
```

## 改进优先级建议

### P0 - 必须解决
1. 添加停用词过滤（简单但有效）
2. 添加基本的中文分词（使用 jieba 或轻量级方案）
3. 添加容错回退机制

### P1 - 重要改进
1. 优化语义相似度计算
2. 根据文本密度动态调整
3. 性能优化（缓存）

### P2 - 可选优化
1. 引入预训练模型（Sentence-BERT）
2. 可配置的切分策略
3. 可视化调试工具

## 结论

当前方案在**大多数场景下可以正常工作**，但在以下场景可能存在问题：
- 专业/技术内容视频
- 文本密度差异很大的视频
- 对切分准确性要求极高的场景

建议先实现**P0级别的改进**（停用词、简单分词、容错机制），这些改进简单但效果显著。
