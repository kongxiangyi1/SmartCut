# 文本切分优化方案

## 当前问题

### 1. 固定 30 分钟切分的问题

| 问题 | 影响 |
|------|------|
| 不管视频长短都用30分钟 | ❌ 短视频也强制切分 |
| 只看停顿时间 | ❌ 没有考虑语义相关性 |
| 忽略LLM token限制 | ❌ 可能超出上下文窗口 |
| 强制切分后备 | ❌ 可能断开话题 |

## 新方案：智能动态切分 ✅

### 核心优化（已实现）

#### 1. **Token 感知切分** - 根据 LLM 上下文窗口动态调整

```python
# 自动检测文本长度，智能调整切分策略
if estimated_tokens > max_tokens_per_chunk * 3:
    # 超长文本 → 缩小到15分钟/块
    interval_minutes = min(interval_minutes, 15)
elif estimated_tokens < max_tokens_per_chunk:
    # 短文本 → 尽量不切分
    interval_minutes = 999
```

#### 2. **语义边界检测** - 识别主题切换的自然边界

```python
# 三级语义检测策略
1. 超长停顿（>5秒）= 高置信度边界 ✅
2. 中等停顿（2-5秒）+ 关键词相似度<30% = 可能的边界
3. 其他停顿 = 普通切分点
```

#### 3. **智能切分日志**

```python
# 优化前
logger.info(f"文本已按~30分钟/块切分，共X个块")

# 优化后
logger.info(f"文本已按智能切分，共X个块（平均约 Y tokens/块）")
# 或
logger.info(f"文本无需切分，单块处理（约 X tokens）")
```

## trust_remote_code 警告解决方案

### 问题说明
```
2026-05-17 18:09:27,465 - root - WARNING - trust_remote_code: False
```

这个警告来自 Hugging Face transformers 库，FunASR 底层使用了 transformers。

### 解决方案

#### 方案1：设置环境变量（推荐）✅

在 `backend/main.py` 开头添加：

```python
import os

# 消除 Hugging Face 警告
os.environ["HF_HUB_DISABLE_SYMLINKS_WARNING"] = "1"
os.environ["TRANSFORMERS_NO_ADVISORY_WARNINGS"] = "1"
os.environ["TOKENIZERS_PARALLELISM"] = "false"
```

#### 方案2：配置 logging（已实现）✅

在 `backend/utils/logging_config.py` 中添加过滤器：

```python
import logging
import os

# 消除 transformers 警告
os.environ["HF_HUB_DISABLE_SYMLINKS_WARNING"] = "1"
os.environ["TRANSFORMERS_NO_ADVISORY_WARNINGS"] = "1"

# 配置 logging 过滤器
class HuggingFaceWarningFilter(logging.Filter):
    def filter(self, record):
        # 过滤掉 trust_remote_code 警告
        if "trust_remote_code" in record.getMessage():
            return False
        return True

# 应用过滤器
logger = logging.getLogger("transformers")
logger.addFilter(HuggingFaceWarningFilter())
```
