# B1 漏洞修复方案：self.llm.chat_one_turn 不存在

## 1. 漏洞确认

**严重性**：运行时错误（AttributeError）

**触发条件**：任意视频进入 `_llm_process_three_step` 流水线，且 Step1.5 gap fill 覆盖率不达标（coverage < 92%）时执行 `_step1_5_gapfill` 方法。

**根本原因**：`FunClipStyleProcessor.__init__` 只初始化了 `self.llm_manager`，没有 `self.llm`，且 `LLMManager` 类没有 `chat_one_turn` 方法。

**验证结果**：
```
self.llm_manager 存在: True
self.llm 存在:        False
llm_manager 有 call(): True
llm_manager 有 chat_one_turn(): False
```

---

## 2. 修复方案

### 2.1 根本修复（推荐）

将 `_step1_5_gapfill` 中第1872行的 LLM 调用从：
```python
# 错误：self.llm 不存在，chat_one_turn 不存在
response = self.llm.chat_one_turn(prompt, temperature=0.3)
```
改为：
```python
# 正确：通过 LLMManager.call() 统一入口
response = self.llm_manager.call(prompt, temperature=0.3)
```

`LLMManager.call()` 返回 `response.content` 即纯文本字符串，与后续的JSON解析逻辑兼容。

### 2.2 关于 `input_data` 参数

观察类中原有其他LLM调用模式：

| 方法 | 调用方式 | 返回类型 |
|------|---------|---------|
| `_call_step3_batch_title` (L1786) | `self.llm_manager.current_provider.call(系统prompt, 输入数据)` | `LLMResponse` → `.content` |
| `_step1_5_gapfill` (待修复) | `self.llm_manager.call(prompt, temperature=0.3)` | `str` |

对于Step1.5场景，`_step1_5_gapfill` 使用 `FUNCLIP_STEP1_5_GAPFILL_PROMPT.format(gap_srt=..., existing_topics=...)` 格式化的完整提示词，**不需要额外的 `input_data` 参数**。

### 2.3 变更范围

| 文件 | 行号 | 修改 |
|------|------|------|
| `funclip_style.py` | L1872 | `self.llm.chat_one_turn` → `self.llm_manager.call` |

**仅1处修改，1行代码。**

## 3. 修复后验证方案

### 3.1 验证方法1：单元测试（推荐优先）

编写一个最小测试，模拟覆盖率不足的场景，验证 `_step1_5_gapfill` 可正常调用LLM：

```python
# backend/tests/test_step1_5_gapfill.py
import pytest
from unittest.mock import MagicMock, patch
from backend.pipeline.funclip_style import FunClipStyleProcessor
from backend.pipeline.funclip_style import _compute_topic_coverage_simple, _get_step1_5_config


def test_step1_5_gapfill_calls_llm_manager():
    """验证 _step1_5_gapfill 使用 self.llm_manager.call() 而非 self.llm.chat_one_turn"""
    proc = FunClipStyleProcessor()

    # 确认 self.llm_manager.call 存在，self.llm 不存在
    assert hasattr(proc, 'llm_manager'), "self.llm_manager 必须存在"
    assert hasattr(proc.llm_manager, 'call'), "llm_manager.call 必须存在"
    assert not hasattr(proc, 'llm'), "self.llm 不应存在（会导致 AttributeError）"

    # mock llm_manager.call 返回一个可解析的JSON
    mock_response = '```json\n[{"outline": "测试话题", "segments": [{"start": "00:00:00,000", "end": "00:00:10,000"}]}]\n```'
    proc.llm_manager.call = MagicMock(return_value=mock_response)

    # 构造覆盖率不足的场景
    config = _get_step1_5_config()
    topic = {
        'outline': '已有话题',
        'segments': [{'start': '00:01:00,000', 'end': '00:02:00,000'}]
    }
    entries = [
        {'start': 0, 'end': 5, 'text': '开头'},
        {'start': 100, 'end': 105, 'text': '末尾'},
    ]

    result = proc._step1_5_gapfill([topic], entries, '')

    # 验证 LLM 被调用
    proc.llm_manager.call.assert_called_once()
    assert len(result) >= 1
```
