## 📅 二期（待开发）

### 预计实施日期：待定

### P1级别改进（优先级：中）

#### 任务1：根据文本密度动态调整
**目标**：根据视频内容密度自动调整切分时长

**详细说明**：
- 计算文本密度（总字数/视频时长）
- 高密度视频（>8字/秒）→ 15分钟块
- 中等密度视频（4-8字/秒）→ 30分钟块
- 低密度视频（<4字/秒）→ 45分钟块

**实现位置**：
```
backend/utils/text_processor.py
新增方法：_calculate_optimal_chunk_duration()
修改方法：_chunk_srt_data_smart() - 在开头调用密度计算
```

**预计工作量**：1-2天

---

#### 任务2：添加缓存机制
**目标**：缓存切分结果，避免重复计算

**详细说明**：
- 基于内容生成缓存键（MD5哈希）
- 缓存切分结果
- 相同内容直接返回缓存

**实现位置**：
```
backend/utils/text_processor.py
新增属性：_chunk_cache
新增方法：_generate_cache_key()
修改方法：chunk_srt_data() - 添加缓存逻辑
```

**预计工作量**：1天

---

### P2级别改进（优先级：低）

#### 任务1：引入 jieba 分词
**目标**：使用 jieba 进行更准确的中文分词

**详细说明**：

- 实现 jieba 分词方法

**依赖**：


**实现位置**：
```

```

**预计工作量**：0.5天

---

#### 任务2：使用 Sentence-BERT
**目标**：使用预训练模型计算更准确的语义相似度

**详细说明**：
- 安装 sentence-transformers 库
- 加载预训练模型
- 计算文本嵌入和余弦相似度
- 嵌入缓存优化

**依赖**：
```
requirements.txt
sentence-transformers>=2.2.0
```

**实现位置**：
```
backend/utils/text_processor.py
新增属性：_model, _embedding_cache
新增方法：_calculate_semantic_similarity_bert(), _get_embedding()
```

**注意事项**：
- 模型较大（约500MB），首次需要下载
- 内存占用增加，需要评估服务器配置

**预计工作量**：1-2天

## 🎯 使用建议

### 当前（一期完成后）
✅ **可以直接使用**，在大多数场景下都能良好工作！

### 二期决策建议
| 场景 | 建议 |
|------|------|
| 处理很多教学视频 | 实施P1-文本密度动态调整 |
| 用户会重复上传同一视频 | 实施P1-缓存机制 |
| 对切分质量要求极高 | 实施P2-jieba分词 |
| 资源受限（内存小） | 不实施P2-Sentence-BERT |
| 生产环境大量用户 | 考虑P1，谨慎P2 |
