# 音频处理切分参数说明

## 问题1：trust_remote_code 警告

### 警告信息
```
2026-05-17 18:09:27,465 - root - WARNING - trust_remote_code: False
```

### 问题分析
这个警告来自 Hugging Face 的 transformers 库，通常在加载某些预训练模型时出现。

### 影响
- **不是错误**，只是一个警告信息
- 可能影响 FunASR 某些模型的使用
- 不会导致系统崩溃或主要功能失效

### 解决方案
如果想消除这个警告，可以在加载模型时设置 `trust_remote_code=True`。但由于我们的 FunASR 使用的是标准模型，这个警告不会造成实质影响。

## 问题2：文本按~30分钟/块切分

### 日志信息
```
2026-05-17 18:10:52,733 - backend.pipeline.step1_outline - INFO - 文本已按~30分钟/块切分，共X个块
```

### 切分原因

#### 1. **LLM 上下文窗口限制**
- 大多数 LLM（如 GPT、Claude、通义千问等）有输入 token 限制
- 通常在 4K-128K tokens 之间
- 一个 30 分钟的视频（约 3000-5000 字）可能接近单个请求的极限
- 切分后每个块约 2000-3000 字，适合 LLM 处理

#### 2. **处理时间和成本优化**
- 单次 API 调用时间过长会增加成本
- 切分后可以并行处理，提高效率
- 降低单次请求失败的风险

#### 3. **Topic 边界的准确性**
- 代码在寻找切分点时，优先选择**停顿处**（≥1秒的间隔）
- 这样可以避免在对话中间断开
- 保持话题的完整性

### 代码实现细节

#### 文件位置
- **主逻辑**: `backend/pipeline/step1_outline.py` 第67行
  ```python
  chunks = self.text_processor.chunk_srt_data(srt_data, interval_minutes=30)
  ```

- **切分算法**: `backend/utils/text_processor.py` 第82行
  ```python
  def chunk_srt_data(self, srt_data: List[Dict], 
                     interval_minutes: int = 30,  # 核心参数！
                     pause_threshold_ms: int = 1000,
                     overlap_minutes: int = 2) -> List[Dict]:
  ```

#### 切分算法特点

1. **智能寻找停顿点**
   - 在目标时间的 90%-110% 范围内寻找 ≥1秒 的停顿
   - 在停顿处切分，保证话题完整性

2. **2分钟重叠机制**
   - 每个块之间有 2 分钟的重叠区域
   - 提供相邻块的上下文，防止话题被切断

3. **强制切分机制**
   - 如果找不到合适的停顿点，在目标时间点强制切分
   - 确保每个块的时间长度相对均匀

### 是否需要调整？

#### 建议保持 30 分钟的理由
1. **中文识别准确率**: 30分钟的视频长度是大多数视频的标准分段
2. **LLM 处理能力**: 约 3000-5000 字，适合大多数 LLM 的上下文窗口
3. **话题完整性**: 智能停顿检测 + 2分钟重叠 = 高质量的话题切分

#### 可以调整的场景
- 如果使用支持更长上下文的 LLM（如 GPT-4 128K、Claude 200K），可以增加到 60-90 分钟
- 如果处理短视频（<30分钟），可以减少到 10-15 分钟

### 如何修改切分时长？

如果确实需要调整，可以修改以下位置：

**方法1**: 修改默认参数（不推荐，影响全局）
```python
# backend/utils/text_processor.py 第82行
def chunk_srt_data(self, srt_data: List[Dict], 
                   interval_minutes: int = 60,  # 改为 60 分钟
                   ...):
```

**方法2**: 通过配置文件控制（推荐）
在 `backend/core/shared_config.py` 中添加：
```python
CHUNK_INTERVAL_MINUTES = 60  # 或其他值
```

然后在调用处使用：
```python
chunks = self.text_processor.chunk_srt_data(srt_data, 
                                           interval_minutes=CHUNK_INTERVAL_MINUTES)
```

**方法3**: 运行时动态调整
在处理不同项目时，根据视频时长自动调整切分策略。
